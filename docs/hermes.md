# Hermes WebUI NixOS Module
This module packages the Hermes WebUI stack as a reproducible NixOS service set built around rootless Podman containers.
It is designed to run these components together:
* `hermes-agent`
* `hermes-webui`
* `hermes-dashboard` when enabled
* `caddy` when enabled
The module lives in [`hermes.nix`](../src/roles/nixos/files/etc/nixos/modules/hermes.nix) and exposes a single NixOS option set at `services.hermes-webui`.
## What The Module Does
* Creates a dedicated system user and group for the Podman runtime when `rootless = true`
* Creates the shared OCI network and named volumes
* Writes Hermes model configuration into the persistent `hermes-home` volume
* Copies a secret environment file into that same persistent volume so Hermes can read API keys at runtime
* Runs Caddy in front of Hermes WebUI and publishes only the Caddy port
## Runtime Layout
The stack uses these persistent objects by default:
* Network: `hermes-net`
* Config/state volume: `hermes-home`
* Agent source volume: `hermes-agent-src`
* Host workspace: `/var/lib/hermes-webui/workspace`
The main files end up here inside the `hermes-home` volume:
* `config.yaml`
* `.env`
Those files are generated by `hermes-webui-oci-setup.service` before the containers start.
## Secrets
Use `services.hermes-webui.environmentFile` for secrets such as:
```
OPENROUTER_API_KEY=sk-or-v1-...
HERMES_WEBUI_PASSWORD=...
HERMES_DEFAULT_MODEL=qwen/qwen3.6-flash
```
The module copies that file into the persistent Hermes home volume as `.env` and makes it readable by the Hermes container user. The file is not written into the Nix store.
The module also reads `HERMES_DEFAULT_MODEL` from that file when present and uses it to populate `config.yaml`. If the variable is absent, the fallback default is:
```
qwen/qwen3.6-flash
```
The generated WebUI runtime environment is kept in sync with the same resolved model so the browser picker starts on the same default instead of the WebUI image's provider fallback.
The provider and base URL are written as:
* Provider: `openrouter`
* Base URL: `https://openrouter.ai/api/v1`
## Container Model
The module is set up for rootless Podman by default:
* Service user: `hermes-webui`
* Service group: `hermes-webui`
* Service UID/GID: `989`
* Container UID/GID: `1000:1000`
That split keeps the host service account separate from the user identity Hermes expects inside its containers.
The containers do not need sudo access or membership in any privileged host group.
## Port Exposure
The recommended setup is to expose only Caddy on the host and keep the app containers private on the internal Podman network.
Relevant options:
* `enableCaddy = true`
* `caddyHostPort = 8787` for the current deployment
* `caddyListenAddress = "0.0.0.0"` if you want it reachable on all interfaces
If you want the conventional HTTP port later, set `caddyHostPort = 80`.
The WebUI itself still listens on `8787` inside the container, but it is not published directly to the host.
## Key Options
* `enable`: turn the stack on
* `backend`: `podman` or `docker`
* `rootless`: run containers as the dedicated service user
* `user` / `group`: host account names for the Podman service user
* `environmentFile`: secret env file containing API keys and runtime secrets
* `defaultModel`: fallback Hermes model
* `defaultModelEnvVar`: env var used to override the model from the secret file
* `inferenceProvider`: written into `config.yaml`
* `baseUrl`: provider URL written into `config.yaml`
* `caddyHostPort`: published host port for the proxy
* `webuiPort`: WebUI container port, default `8787`
* `gatewayPort`: Hermes gateway port, default `8642`
* `dashboardPort`: Hermes dashboard port, default `9119`
## Operational Notes
* `hermes-webui-oci-setup.service` is responsible for creating the network, volumes, and generated config files.
* `podman-hermes-webui.service` depends on the setup unit and the agent container.
* `podman-hermes-caddy.service` depends on the WebUI container.
* If you change the secret file, restart the setup unit and then restart the Hermes containers so they reload the copied `.env`.
## Example NixOS Configuration
```
{ services.hermes-webui = { enable = true; rootless = true; environmentFile = "/run/secrets/hermes-webui.env"; caddyHostPort = 8787; defaultModel = "qwen/qwen3.6-flash"; inferenceProvider = "openrouter"; baseUrl = "https://openrouter.ai/api/v1"; };
}
```
## Security Model
The module is intended to keep the Hermes containers unprivileged:
* dedicated host user and group
* no sudo access
* no extra host group membership
* secret material injected through a root-owned env file
* all app containers isolated on a private Podman network
The main host-level exposure is the chosen Caddy port.
