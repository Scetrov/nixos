## ADDED Requirements

### Requirement: Grafana OAuth client ID consistency
Grafana OAuth configuration SHALL use the Authentik Grafana OAuth2 provider client ID generated by OpenTofu and deployed through the generated-secret and agenix workflow.

#### Scenario: Grafana redirects with current Authentik client ID
- **WHEN** a user starts Grafana login through `/grafana/login/generic_oauth`
- **THEN** the `client_id` in the Authentik authorization request matches the current OpenTofu-managed Authentik Grafana OAuth2 provider client ID

#### Scenario: Grafana rejects placeholder OAuth configuration
- **WHEN** the generated Grafana OAuth client ID is empty, missing, or contains a placeholder value
- **THEN** the deployment fails before Grafana service configuration is applied

#### Scenario: Grafana service receives refreshed OAuth environment
- **WHEN** the generated Grafana Authentik client ID or client secret changes
- **THEN** Grafana receives the updated systemd environment file and is restarted through the IaC deployment flow
